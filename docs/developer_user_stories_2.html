508 Compliant PDF Conversion Agent Developer User Stories: Sprint 2

Ticket: LIN-006 - Integrate RouteLLM

Description: Integrate RouteLLM into the existing system for routing requests between strong and weak models based on the size of PDF files.

Acceptance Criteria:

  • RouteLLM is set up and configured for the project.
  • Requests are routed appropriately based on the PDF file size.
  • Google's Gemini-Pro is used for large files, and OpenAI's GPT-4o-mini is used for smaller files.
  • Integration is tested to ensure correct routing and response handling.

Ticket: LIN-007 - Implement Model Selection Logic

Description: Implement the logic within RouteLLM to determine which LLM model to use based on the size of the PDF file.

Acceptance Criteria:

  • Model selection is based on predefined thresholds for file size.
  • Ensure the system can dynamically select between Gemini-Pro and GPT-4o-mini.
  • Test cases confirm that the correct model is selected for a range of file sizes.

Ticket: LIN-008 - Testing and Validation of RouteLLM Integration

Description: Conduct thorough testing and validation of the RouteLLM integration, ensuring proper routing and accurate response generation.

Acceptance Criteria:

  • Test with a variety of PDF file sizes to ensure proper routing and model selection.
  • Verify that the outputs from both Gemini-Pro and GPT-4o-mini meet the system prompt requirements.
  • Document any issues and ensure they are resolved before the end of the sprint.
